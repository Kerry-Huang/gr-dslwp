13 /* This parameter sizes the path memory in bits, which is organized as a
14  * circular buffer through which we periodically "trace back" to
15  * produce the decoded data. PATHMEM must be greater than
16  * MERGEDIST+TRACECHUNK, and for efficiency it should also be a power of 2.
17  * Don't make it *too* large, or it will spill out of the CPU's on-chip cache
18  * and decrease performance. Each bit of path memory costs 8 bytes for the
19  * K=7 code.
20  */
21 #define PATHMEM 128
22 
23 /* In theory, a Viterbi decoder is true maximum likelihood only if
24  * the path memory is as long as the entire message and a single traceback
25  * is made from the terminal state (usually zero) after the entire message
26  * is received.
27  *
28  * In practice, performance is essentially optimum as long as decoding
29  * decisions are deferred by at least 4-5 constraint lengths (28-35 bits
30  * for K=7) from the most recently received symbols. MERGEDIST sets this
31  * parameter. We give ourselves some margin here in case the code is
32  * punctured (which slows merging) and also to let us start each traceback
33  * from an arbitrary current state instead of taking the time to find the
34  * path with the highest current metric.
35  */
36 #define MERGEDIST 64 /* Distance to trace back before decoding */
37 
38 /* Since each traceback is costly (thanks to the overhead of having to
39  * go back MERGEDIST bits before we produce our first decoded bit) we'd like
40  * to decode as many bits as possible per traceback at the expense of
41  * increased decoding delay. TRACECHUNK sets how many bits to
42  * decode on each traceback. Since output is produced in 8-bit bytes,
43  * TRACECHUNK MUST be a multiple of 8.
44  */
45 #define TRACECHUNK 8 /* How many bits to decode on each traceback */
46 
47 /* The path metrics need to be periodicially adjusted downward
48  * to prevent an integer overflow that could cause the signed comparisons
49  * in the butterfly macros to fail.
50  *
51  * It's possible to code the comparisons to work in modulo fashion, e.g.,
52  * as 'if((a-b) > 0)' rather than 'if(a >b)'. A good optimizer would generate
53  * code like 'cmp a,b;js foo' for this, but GCC doesn't.
54  *
55  * This constant should be larger than the maximum path metric spread.
56  * Experimentally this seems to be 2040, which is probably related to the
57  * free distance of the code (10) and the symbol metric scale (0-255).
58  */
59 #define RENORMALIZE 10000
